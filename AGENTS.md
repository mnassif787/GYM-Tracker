## Tech Stack

| Layer | Choice |
|---|---|
| Framework | React 18 + TypeScript 5.7, Vite 5 |
| UI | TailwindCSS 3, Radix UI primitives, shadcn/ui pattern |
| State | Single `WorkoutContext` (no Redux/Zustand) |
| Routing | React Router v6, `BrowserRouter` |
| Forms | `react-hook-form` v7 + `zod` v3 + `@hookform/resolvers` |
| Persistence | IndexedDB via `idb` v8 (DB: `gym-tracker`, schema v2) |
| Testing | Vitest 2 + `@testing-library/react` + `jsdom` + `fake-indexeddb` |
| PWA | `vite-plugin-pwa` with Workbox `autoUpdate` |

## Architecture

```
main.tsx → BrowserRouter → ThemeProvider → WorkoutProvider → App (Routes) → Pages
```

- **Pages** (`src/pages/`) consume `useWorkout()` — never call DB functions directly.
- **`WorkoutContext`** (`src/context/WorkoutContext.tsx`) is the single source of truth for all runtime state. All mutations (`startWorkout`, `addSet`, `completeWorkout`, `deleteWorkout`) are here.
- **`src/lib/db.ts`** — six thin async wrappers over IndexedDB; module-level singleton `dbPromise` (lazy init). Tests must call `vi.resetModules()` in `beforeEach` to reset it.
- **`src/lib/exercises.ts`** — `DEFAULT_EXERCISES` array is the authoritative exercise list; DB overrides are merged on top at load time. A new exercise ID not in `DEFAULT_EXERCISES` is silently dropped.

## Key Conventions

- **Path alias**: always `@/` → `src/`, never relative imports
- **Exports**: named exports everywhere; only `App` and `main` use default export
- **Forms**: every form uses `react-hook-form` + `zodResolver` + inline `z.object()` schema at file top + `type X = z.infer<typeof xSchema>`
- **Toasts**: use `sonner`'s `toast.success/error/info` — no `alert()`, no inline error state for async ops
- **UI components**: live in `src/components/ui/` (shadcn-style primitives); feature components in `src/components/`
- **Local sub-components** (e.g., `StatCard`) go at the bottom of the same file, not in separate files

## GitHub Workflow Notes

No CI is configured yet (no `.github/` folder). When setting up GitHub Actions:

- **CI command**: `npx vitest run` (single-pass, not watch) + `npm run build`
- **Node version**: use the version from `package.json` engines if added, otherwise Node 20 LTS is safe with this stack
- **Build artifact**: `dist/` — generated by Vite, suitable for GitHub Pages or any static host
- **Branch strategy**: no conventions enforced yet; suggest `main` as default + feature branches

## Known Pitfalls

1. **`getRecommendedWeight` always adds +2.5** regardless of unit (`kg`/`lbs`) — if touching weight logic, fix the unit check.
2. **Exercise creation at runtime is broken** — new exercise IDs not in `DEFAULT_EXERCISES` are silently dropped on next load. Must add to `DEFAULT_EXERCISES` first.
3. **Admin auth is a demo stub** — hardcoded password in `WorkoutContext.tsx`, stored in `localStorage`. Comment says `// INSECURE: demo only`.
4. **`weightUnit` is duplicated** on both `Exercise` and `Workout` — historical workouts retain the unit at log time; no migration runs on unit change.
5. **`dbPromise` singleton** breaks tests if `vi.resetModules()` is not called in `beforeEach` of db-specific tests.

## Testing Pattern for DB Code

```typescript
beforeEach(async () => {
  globalThis.indexedDB = new IDBFactory(); // fresh fake-indexeddb instance
  vi.resetModules();                        // reset dbPromise singleton
  const { functionUnderTest } = await import('@/lib/db');
  // ...
});
```
